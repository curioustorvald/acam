This webapp implements my theory about the Artistic Colour Appearance Model.

It is a common tip in the art community that when you use HSV colour model for your colour picker, vary the H-slider as you darken the colour to make it look more vibrant. The ACAM (pronounced /'æɪ.kæm/) generalises the idea using the single parameter Variance over the LChUV colour space.

How to Use

Before you start, select the right RGB Model. If you don’t know what RGB Model is, simply select the sRGB; if you’re using Apple devices, select Display P3.

First, select the Eigencolour. The Eigencolour is the true colour of the object.

Second, adjust the Variance slider so that the colour gradient look vibrant enough.

Third, select the Ambient Colour. The Ambient Colour is the colour of the ambient light. Also adjust the Ambient Mix as you need.

Last, pick the desired colour by simply clicking and dragging on the gradient square. The Hex Code can be copied by clicking on the Selected Colour swatch.

Troubleshooting

  • Selected Colour Circle is offsetted from where I’ve touched!
    ☞ Simply refresh the browser.
  • Selected Colour’s Hex Code is Red!
    ☞ Red text means the colour is out of gamut of the selected RGB Model. This is mostly harmless; it just means the selected colour can’t be represented accurately due to the limit of the RGB itself. Again, this is harmless.
